# API документация
**Базовый URL:** `http://localhost:5000/api/v1`  
**Формат данных:** JSON  
**Access токен:** Передаётся в заголовке `Authorization: Bearer <access_token>`  
**Refresh токен:** Хранится в httpOnly cookie, автоматически отправляется браузером

---

## 1) auth

### 1.1 POST /auth/register
**Доступ:** Публичный  
**Описание:** Регистрация нового клиента  
**Тело запроса:**

| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| email | string | Да | Электронная почта пользователя |
| password | string | Да | Пароль (минимум 6 символов) |
| fullName | string | Да | Полное имя пользователя |

**Пример запроса:**

```json
{
  "email": "ivanov@example.com",
  "password": "password123",
  "fullName": "Иванов Иван Иванович"
}
```

**Пример успешного ответа (201 Created):**
```json
{
  "access_token": "eyJhbGciOiJIUzI1NiIs...",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "email": "ivanov@example.com",
    "fullName": "Иванов Иван Иванович",
    "role": "client"
  }
}
```

**Пример ошибки (400 Bad Request):**
```json
{
  "error": "Пользователь с таким email уже существует"
}
```

**Cookie (устанавливается автоматически):**
| Ключ | Значение | 
|------|-----|
| refresh_token | httpOnly, secure, sameSite=strict |

**Описание:** Вход в систему  
**Тело запроса:**
| Поле | Тип | Обязательное | Описание |
|------|-----|--------------|----------|
| email | string | Да | Электронная почта пользователя |

**Пример запроса:**

**Пример успешного ответа (200 OK):**

  "access_token": "eyJhbGciOiJIUzI1NiIs...",

**Cookie (устанавливается автоматически):**
| Ключ | Значение | 
|------|-----|
| refresh_token | httpOnly, secure, sameSite=strict | 

---
### 1.3 GET /auth/refresh
**Доступ:** Публичный  
**Описание:** Получение нового access токена по refresh токену из cookie  
**Cookie (отправляется автоматически):**
| Ключ | Значение | 
|------|-----|
| refresh_token | httpOnly cookie, полученный при логине или регистрации | 
**Пример успешного ответа (200 OK):**
```json
{
  "access_token": "new_jwt_token",
  "token_type": "bearer"
}
```
**Пример ошибки (401 Unauthorized):**
```json
{
  "error": "Refresh токен не найден или истёк"
}
```
---
### 1.4 GET /auth/me
**Доступ:** Авторизованный (любая роль)  
**Описание:** Получение информации о текущем пользователе  
**Заголовки:**
| Ключ | Значение | 
|------|----------|
| Authorization | Bearer <token> |

**Пример успешного ответа (200 OK):**
```json
{
  "id": 1,
  "email": "ivanov@example.com",
  "fullName": "Иванов Иван Иванович",
  "role": "client"
}
```

**Пример ошибки (401 Unauthorized):**
```json
{
  "error": "Токен не предоставлен"
}
```

### 1.5 POST /auth/logout
**Доступ:** Авторизованный  
**Описание:** Выход из системы. Удаляет refresh токен из cookie и аннулирует его на сервере.  
**Заголовки:**
| Ключ | Значение | 
|------|----------|
| Authorization | Bearer <token> |
**Пример успешного ответа (200 OK):**
```json
{
  "message": "Вы успешно вышли из системы"
}
```
**Результат:** Cookie `refresh_token` удаляется (устанавливается с истекшим сроком действия).
